# KRNO IMP — دليل التشغيل

## الملفات الموجودة
- `import-system.html` — التطبيق نفسه
- `manifest.json` — ملف PWA (يخلي التطبيق قابل للتثبيت كأيقونة)
- `service-worker.js` — يخلي التطبيق يفتح بدون شريط المتصفح
- `icon-192.png` / `icon-512.png` — أيقونة التطبيق
- `krno_import_schema.sql` — كود قاعدة البيانات (تشغيل مرة واحدة فقط)

**مهم:** الخمس ملفات لازم تكون في نفس الفولدر مع بعض على الاستضافة (Netlify/GitHub Pages)، لأن `manifest.json` و `service-worker.js` بيتم تحميلهم بمسار نسبي.

---

## الخطوة 1 — تشغيل كود قاعدة البيانات

1. افتح Supabase Dashboard بتاع مشروعك الموجود (نفس بتاع المخازن)
2. من القائمة الجانبية: **SQL Editor** → **New query**
3. افتح ملف `krno_import_schema.sql`، انسخ كل المحتوى، الصقه، ودوس **Run**
4. هيتم إنشاء سكيمة جديدة اسمها `krno_import` فيها كل الجداول، بدون أي تعارض مع باقي السكيمات

## الخطوة 2 — كشف السكيمة للـ API

1. من القائمة الجانبية: **Project Settings** → **API**
2. لاقي قسم **Exposed schemas**
3. ضيف `krno_import` في القائمة (public موجودة بالفعل، أضف دي جنبها)
4. احفظ

## الخطوة 3 — نسخ بيانات الاتصال

1. في نفس صفحة **Project Settings** → **API**
2. انسخ:
   - **Project URL** (شكلها `https://xxxxxxxxx.supabase.co`)
   - **anon public key** (مفتاح طويل تحت "Project API keys")

## الخطوة 4 — لصق البيانات في التطبيق

1. افتح `import-system.html` بأي محرر نصوص (Notepad أو VS Code)
2. دوس Ctrl+F وابحث عن: `YOUR_SUPABASE_PROJECT_URL`
3. استبدلها بالـ Project URL بتاعك (مع علامات التنصيص حواليها)
4. ابحث عن `YOUR_SUPABASE_ANON_KEY` واستبدلها بالـ anon public key
5. احفظ الملف

مثال بعد التعديل:
```js
const SUPABASE_URL = 'https://abcdefgh.supabase.co';
const SUPABASE_ANON_KEY = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...';
```

## الخطوة 5 — رفع الملفات

ارفع الخمس ملفات مع بعض على Netlify أو GitHub Pages (زي باقي مشاريعك). بعد الرفع:

- افتح الرابط من الموبايل → في كروم: ⋮ → "Add to Home screen"
- افتح الرابط من اللابتوب → في كروم/إيدج: أيقونة التثبيت ⊕ في شريط العنوان

التطبيق هيفتح بعد كذلك كأيقونة منفصلة بدون شريط المتصفح، على الموبايل والديسكتوب.

## تسجيل الدخول لأول مرة

نفس المستخدمين التجريبيين، لكن دلوقتي محفوظين في قاعدة البيانات (`krno_import.app_users`):
- `admin` / `admin123` — مدير نظام
- `khaled` / `khaled1` — مشرف
- `viewer` / `view1` — مشاهدة فقط

**ينصح بتغيير كلمات السر** من Supabase Table Editor → `krno_import` → `app_users` بعد أول استخدام.

## التحديث الفوري بين الأجهزة

بعد ضبط الخطوات اللي فوق، أي إضافة أو تعديل أو حذف (فاتورة، دفعة، مصروف) هيظهر فورًا على كل جهاز فاتح التطبيق في نفس اللحظة — مفيش حاجة تانية تعملها، التزامن شغال تلقائي.

في شريط العنوان جوه التطبيق هيظهر:
- 🟢 **متصل — تحديث فوري** = كل حاجة تمام
- ⚪ **غير متصل (وضع محلي)** = لسه مفيش بيانات اتصال، أو في مشكلة في الشبكة — البيانات هتتسجل مؤقتًا في الجهاز نفسه بس من غير مزامنة
